## Overview

I developed a 2D Role-Playing Game from scratch to challenge my understanding of object-oriented programming and software architecture. Instead of just relying on a game engine's built-in tools, I focused heavily on designing the underlying logic—such as character progression, stat calculations, and dynamic combat interactions—ensuring a smooth and mathematically balanced gameplay experience.

## Key Engineering Features

- **Combat & Stat Calculations**: Implemented a robust combat system with precise mathematical models for damage output, defense mitigation, and complex skill buffs. 
- **State Management**: Designed an efficient state machine to handle character animations, active skills, and scene transitions seamlessly.
- **Object-Oriented Design**: Structured the codebase using strong OOP principles (inheritance, polymorphism) to manage diverse enemy types, inventory items, and NPC behaviors efficiently.
- **Collision & Physics**: Programmed accurate hitboxes and collision detection algorithms to ensure responsive and fair gameplay mechanics.

## Technologies Used

- **C++**: Core game logic and mathematics.
- **Massey Game Engine**: Rendering and input handling.
- **Git & GitHub**: Version control and code management.

## Challenges and Learnings

One of the most complex challenges was balancing the game's economy and combat math. Writing the logic for conditional stat multipliers and ensuring memory efficiency when multiple objects were rendered simultaneously deeply improved my problem-solving skills. I learned how to structure scalable code that can handle real-time updates without dropping frame rates.
